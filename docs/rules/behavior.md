# 行为规则

AGENTS.md 中的"行为底线"是每次任务必须遵守的最小集合。本文件是完整版，按需加载。

---

# 思考方式

- **修改前先理解**：阅读相关代码，理解模块职责、调用关系、数据流向和已有模式
- **禁止猜测**：行为未明确时继续搜索——读调用方、读测试、读类型定义，直到确认
- **新增前先搜索**：已有实现、工具、类型、组件、Hook/Service/Repository

---

# 工作原则

- **跟随项目**：优先遵循项目已有模式。即使存在理论上更优方案，不擅自替换。代码应"看起来本来就属于这里"
- **控制复杂度**：不为假想未来需求提前抽象。删掉比增加更重要
- **渐进修改**：分小步完成，每步保持系统可运行。禁止一次性推翻重写
- **优先复用**：满足需求则复用，禁止仅因偏好重复实现
- **伪完成不算完成**：不留未完成逻辑，不用 TODO 代替核心实现，不用 mock 伪装完成

---

# 场景原则

- **Debug**：先观察现象定位根因，再修复。禁止一次改多个变量
- **重构**：必须有测试覆盖，一次只做一件事，前后行为不变
- **性能优化**：先测量确认瓶颈，每次只优化一个点，不为小收益牺牲可读性
- **高风险区域**（数据库 Schema、公共组件、认证逻辑）：搜索全部引用，评估影响范围，确认回退方案
- **测试**：Bug 修复必须补充回归测试防止复发。新功能按项目已有测试风格补充

---

# 决策优先级

冲突时遵循：正确性 > 可维护性 > 项目一致性 > 可读性 > 性能 > 理论优雅性
