# 规则哲学

本文件定义规则系统自身的设计原则。

# 一、行为约束 > 技术规范

AI 犯错极少因为“不会写代码”，而是乱猜、乱改、蔓延、过度工程化。
规则的第一任务是防错，第二才是教写法。

# 二、塑造判断力，不下达指令

目标是让 AI 像资深工程师一样思考，不是把它变成脚本。
好的规则提高理解和判断力，坏的规则增加机械限制。

# 三、上下文优先于教条

跟随现有项目，保持一致性，尊重历史架构。
优先定义“优先最小修改”等原则，而非“必须用 xxx 库”等实现细节。

# 四、克制

- **默认 > 强制**：只有真正危险才用【强制】，强制过多压制推理。
- **只解决真实问题**：不凭空设计规则。
- **删比增更重要**：规则越多，冲突越重，必须持续删减。

# 五、简洁即力量

每条规则：短、明确、高信息密度。
AGENTS.md 必须极短，只放行为底线和索引，具体规则按需放在 `docs/rules/` 下。
不允许一个超级大规则文件。
